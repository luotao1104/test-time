# AODW 治理检查

检测 AODW 规则和流程中的问题，提议改进。

## 功能
- 检测规则不清楚的情况
- 检测缺少文档模式
- 检测重复的用户困惑
- 检测规则执行中的冲突
- 检测模块缺少 README 但被引用
- 检测 RT 流程变得模糊
- 提议创建 RT-CORE 来增强 AODW

## 检测场景
- 规则不清楚
- 缺少文档模式
- 重复的用户困惑
- 规则在执行中引起冲突
- 模块缺少 README 但被引用
- RT 流程变得模糊

## 提议流程
当检测到治理问题时：
1. 询问用户是否创建 RT-CORE
2. 如果同意，创建 RT/RT-CORE-XXX/
3. 遵循 Spec-Full 流程
4. 更新治理版本（如应用更改）

## 使用场景
- 定期检查 AODW 流程健康度
- 发现流程问题时提议改进
- 更新 AODW 治理规则
